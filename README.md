# Mini-Bioinformatics Pipeline & Reporting

This reproducible bioinformatics pipeline performs Quality Control (QC) on raw long-read sequencing data. It calculates fundamental metrics like GC content, read length, and mean quality scores, generating visualizations and summary statistics.

## Prerequisites
- **Conda** (Miniconda or Anaconda) must be installed.
- **Snakemake** should be available in your base environment to launch the pipeline using conda.

## Setup
You can either run this using the built-in Conda integration in Snakemake, or install the environment manually:

```bash
# Optional: create environment manually
conda env create -f environment.yml
conda activate bioinformatics_qc
```

## Running the Pipeline
To run the pipeline and ensure all rules are executed cleanly using conda:
```bash
snakemake --use-conda --cores 1
```
*Note: Make sure your `config.yaml` points to the correct `fastq_file` (e.g., `barcode77.fastq.gz`).*

## Output Files
The pipeline produces multiple outputs in the `results/` directory:
- `results/nanoplot_out/`: Contains the interactive HTML report and graphs generated by NanoPlot.
- `results/metrics_summary.csv`: A structured CSV file with GC Content, Read Length, and Mean Quality metrics for *every* individual read.
- `results/visuals_plots.png`: High-quality histograms/KDE plots illustrating the distribution of the metrics.
- `results/visuals_summary_statistics.txt`: A summary text file noting Mean, Median (50%), Minimum, and Maximum values for the distributions.
